# Test Latent Commands

`IAutomationLatentCommand` subclasses that NEXUS tests queue with `ADD_LATENT_AUTOMATION_COMMAND` to step through fixture setup, frame-by-frame waits, performance bracketing, and teardown without blocking the automation thread. Most callers reach these indirectly through [Test Utils](../test-utils.md), but the building blocks are exposed for tests that need a custom sequence. Mirrors the layout of `NexusCore/Public/Developer/TestLatentCommands/`.
